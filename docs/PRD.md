## 1. 产品概述

### 1.1 产品定位

MobTestLab 是一款面向移动端测试与开发场景的桌面应用工具，基于 Vite + Vue3 + Electron 技术栈构建，兼容 Android、iOS、HarmonyOS 三大移动操作系统。产品定位为**一站式移动端测试工作台**，将设备管理、手机投屏、实时交互、性能测试、UI 自动化测试五大核心能力整合于统一界面。

### 1.2 目标用户

| 用户角色 | 核心诉求 | 使用场景 |
|---------|---------|---------|
| QA 测试工程师 | 高效执行功能测试、回归测试、性能基线对比 | 日常测试执行、自动化用例管理、缺陷复现 |
| 移动端开发工程师 | 快速调试、性能分析、多设备验证 | 开发自测、性能优化、多机型适配验证 |
| 测试开发工程师 | 编写和维护自动化测试脚本 | 自动化框架搭建、CI/CD 集成 |
| 技术管理者 | 测试效率数据、质量报告 | 质量度量、团队效能分析 |

### 1.3 核心价值

- **统一工作台**：告别多工具切换（scrcpy + PerfDog + Appium + ...），一个应用覆盖全流程
- **三端兼容**：Android / iOS / HarmonyOS 统一操作体验
- **AI 驱动测试**：基于 Midscene 的自然语言 UI 自动化，降低自动化门槛
- **实时性能洞察**：毫秒级性能数据采集与可视化，支持历史对比与报告导出

### 1.4 产品边界

**包含**：
- 本地设备连接与管理（USB / WiFi）
- 实时投屏与远程交互
- 应用性能数据采集与分析
- AI 驱动的 UI 自动化测试
- 测试数据持久化与报告导出

**不包含**：
- 云端设备农场（远程真机）
- APP 构建与打包
- 代码级调试（IDE 功能）
- 服务端/接口测试

## 2. 技术架构

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        Electron Shell                            │
├─────────────────────────────────────────────────────────────────┤
│  Main Process (Node.js)                                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ Device   │ │ Screen   │ │ Perf     │ │ Midscene Engine  │  │
│  │ Manager  │ │ Mirror   │ │ Collector│ │ (AI Automation)  │  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────────────┘  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                       │
│  │ ADB      │ │ HDC      │ │ iOS      │                       │
│  │ Bridge   │ │ Bridge   │ │ Bridge   │                       │
│  └──────────┘ └──────────┘ └──────────┘                       │
├─────────────────────────────────────────────────────────────────┤
│  Preload Scripts (contextBridge IPC)                            │
├─────────────────────────────────────────────────────────────────┤
│  Renderer Process (Vue3 + Vite)                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ Device   │ │ Mirror   │ │ Perf     │ │ Automation       │  │
│  │ Panel    │ │ Canvas   │ │ Dashboard│ │ Studio           │  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────────────┘  │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ Shared: Pinia Store / Vue Router / Element Plus UI       │  │
│  └──────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 进程模型与通信

| 进程 | 职责 | 技术栈 |
|------|------|--------|
| Main Process | 设备通信、子进程管理、文件 I/O、系统 API 调用 | Node.js + child_process + net |
| Renderer Process | UI 渲染、用户交互、数据可视化 | Vue3 + Vite + Canvas/WebGL |
| Preload Script | 安全桥接 Main ↔ Renderer 通信 | contextBridge + ipcRenderer |

**IPC 通信机制**：

```
Renderer ──ipcRenderer.invoke()──→ Main Process
Renderer ←──event.sender.send()── Main Process (推送)
Renderer ←──BroadcastChannel────→ Renderer (跨窗口)
```

- 投屏视频流：Main Process 接收原始帧 → SharedArrayBuffer → Renderer 解码渲染
- 性能数据：Main Process 定时采集 → IPC 推送 → Renderer 实时图表更新
- 设备事件：Main Process 监听设备插拔 → IPC 广播 → 全局状态更新

### 2.3 数据存储架构

| 数据类型 | 存储方案 | 说明 |
|---------|---------|------|
| 应用配置 | electron-store (JSON) | 用户偏好、窗口状态、快捷键 |
| 性能测试数据 | SQLite (better-sqlite3) | 历史记录、会话数据、指标时序 |
| 测试脚本 | 本地文件系统 (.mts 文件) | 自然语言测试用例 |
| 测试报告 | HTML/PDF 模板导出 | 基于 ECharts 图表的可视化报告 |
| 设备信息缓存 | LevelDB / JSON | 已连接设备的元数据缓存 |

## 3. 功能模块设计

### 3.1 设备管理模块

#### 3.1.1 功能概述

统一管理 Android、iOS、HarmonyOS 三大平台设备的发现、连接、状态监控。

#### 3.1.2 功能需求

**FR-DM-001 设备自动发现**

| 平台 | 发现机制 | 连接方式 |
|------|---------|---------|
| Android | ADB daemon 监听 USB/TCP | USB 数据线 / WiFi (adb tcpip) |
| iOS | usbmuxd 服务监听 | USB 数据线 / WiFi (需同网段配对) |
| HarmonyOS | HDC daemon 监听 | USB 数据线 / WiFi (hdc tconn) |

- 应用启动时自动扫描已连接设备
- 支持 USB 热插拔实时检测（轮询间隔 ≤ 2s）
- 支持手动输入 IP:Port 连接远程设备
- 设备连接/断开时发送系统通知

**FR-DM-002 设备信息展示**

每台设备展示以下信息：

| 信息项 | Android | iOS | HarmonyOS |
|--------|---------|-----|-----------|
| 设备名称 | ro.product.model | DeviceName | 设备型号 |
| 系统版本 | ro.build.version.release | ProductVersion | 系统版本 |
| 屏幕分辨率 | wm size | 设备属性 | 屏幕参数 |
| CPU 架构 | ro.product.cpu.abi | uname | CPU 信息 |
| 可用存储 | df /data | 磁盘信息 | 存储信息 |
| 电池电量 | dumpsys battery | 电池信息 | 电池信息 |
| 连接状态 | online/offline/unauthorized | 已配对/未配对 | 已连接/未连接 |

**FR-DM-003 设备操作**

- 一键安装/卸载 APK / IPA / HAP 应用包
- 一键截图并保存到本地
- 文件推送/拉取（设备 ↔ 电脑）
- 重启设备
- 打开/关闭开发者选项相关设置

**FR-DM-004 多设备并行管理**

- 支持同时连接多台设备（上限 8 台）
- 设备列表支持分组（按平台、按项目）
- 支持设备收藏/置顶
- 设备离线后保留历史记录，重新连接时自动恢复会话

### 3.2 一键投屏模块

#### 3.2.1 功能概述

实现手机屏幕实时镜像到桌面应用窗口，支持高帧率、低延迟的画面传输，为后续交互操作和 UI 自动化提供视觉基础。

#### 3.2.2 技术方案

| 平台 | 投屏方案 | 编码格式 | 传输协议 | 预期延迟 |
|------|---------|---------|---------|---------|
| Android | scrcpy-server (自定义集成) | H.264/H.265 | ADB Socket 隧道 | ≤ 70ms |
| iOS | WebDriverAgent MJPEG / pymobiledevice3 QuickTime | MJPEG / H.264 | HTTP Stream / USB | ≤ 100ms |
| HarmonyOS | HDC 截屏轮询 + 增量传输 | PNG/JPEG → WebP | HDC Socket | ≤ 150ms |

#### 3.2.3 功能需求

**FR-SM-001 一键启动投屏**

- 设备列表中点击「投屏」按钮即可启动
- 首次连接 Android 设备时自动推送 scrcpy-server.jar
- iOS 设备自动检测 WDA 是否已安装，未安装时提示引导
- HarmonyOS 设备自动检测 HDC 连接状态
- 投屏启动耗时 ≤ 3s（设备已就绪状态下）

**FR-SM-002 画面质量控制**

| 参数 | 可选值 | 默认值 | 说明 |
|------|--------|--------|------|
| 分辨率 | 原始 / 1080p / 720p / 480p | 720p | 降低分辨率可减少带宽 |
| 帧率 | 60fps / 30fps / 15fps | 30fps | 高帧率适合动画测试 |
| 码率 | 2Mbps / 4Mbps / 8Mbps / 16Mbps | 8Mbps | 影响画面清晰度 |
| 编码 | H.264 / H.265 | H.264 | H.265 更高效但兼容性略低 |

- 支持运行时动态调整参数，无需重启投屏
- 网络带宽不足时自动降级画质（WiFi 连接场景）

**FR-SM-003 投屏窗口管理**

- 支持投屏窗口独立弹出（脱离主窗口）
- 支持窗口自由缩放，保持设备屏幕比例
- 支持全屏模式
- 支持多设备同时投屏（多窗口并排）
- 支持画面旋转（跟随设备方向 / 手动锁定）
- 支持画面截图（快捷键 Ctrl/Cmd + S）
- 支持录屏（MP4 格式，可设置时长上限）

**FR-SM-004 投屏状态监控**

- 实时显示当前帧率、延迟、码率
- 连接断开时自动重连（最多重试 3 次）
- 异常状态提示（设备锁屏、USB 断开、授权失效）

### 3.3 实时交互模块

#### 3.3.1 功能概述

在投屏画面上实现与真实手机一致的交互体验，支持触控、键盘输入、手势操作，让用户无需触碰手机即可完成所有操作。

#### 3.3.2 交互映射方案

| 桌面操作 | 手机动作 | Android 实现 | iOS 实现 | HarmonyOS 实现 |
|---------|---------|-------------|---------|---------------|
| 鼠标左键点击 | 单击 | `adb shell input tap x y` | WDA tap | `hdc shell uitest click x y` |
| 鼠标左键拖拽 | 滑动 | `adb shell input swipe x1 y1 x2 y2 duration` | WDA swipe | `hdc shell uitest swipe` |
| 鼠标滚轮 | 上下滚动 | `adb shell input roll dx dy` | WDA scroll | `hdc shell uitest scroll` |
| 鼠标右键 | 返回键 | `adb shell input keyevent 4` | — | `hdc shell uitest back` |
| 键盘输入 | 文字输入 | `adb shell input text` / IME | WDA typeText | `hdc shell uitest inputText` |
| Ctrl/Cmd+C | 复制 | 剪贴板同步 | 剪贴板同步 | 剪贴板同步 |
| Ctrl/Cmd+V | 粘贴 | 剪贴板注入 | 剪贴板注入 | 剪贴板注入 |
| 双指缩放手势 | 缩放 | 多点触控注入 | WDA pinch | 多点触控注入 |

#### 3.3.3 功能需求

**FR-IA-001 触控交互**

- 鼠标坐标自动映射到设备屏幕坐标（考虑缩放比例和偏移）
- 触控响应延迟 ≤ 50ms
- 支持长按操作（鼠标按住 > 500ms 触发）
- 支持多点触控模拟（Alt + 鼠标拖拽 = 双指缩放）
- 支持自定义手势录制与回放

**FR-IA-002 键盘输入**

- 支持英文、数字、符号直接输入
- 支持中文输入（通过剪贴板注入方式）
- 支持功能键映射：

| 快捷键 | 功能 |
|--------|------|
| Esc | 返回键 (Back) |
| Home (Fn+←) | 主页键 (Home) |
| End (Fn+→) | 最近任务 (Recent) |
| F11 | 音量+ |
| F12 | 音量- |
| Ctrl/Cmd+P | 电源键 |
| Delete | 删除字符 |

**FR-IA-003 剪贴板同步**

- 电脑复制 → 手机粘贴（自动同步）
- 手机复制 → 电脑粘贴（需设备端支持）
- 支持文本和 URL 类型
- 同步延迟 ≤ 200ms

**FR-IA-004 高级交互**

- 支持 GPS 位置模拟（输入经纬度）
- 支持屏幕旋转控制
- 支持息屏/亮屏控制
- 支持通知栏快速下拉
- 支持快速输入预设文本（地址、手机号等测试数据模板）

### 3.4 性能测试模块

#### 3.4.1 功能概述

实时采集目标应用的 CPU、内存、FPS、网络流量、电池、GPU、温度等性能指标，支持可视化展示、历史对比、阈值告警和报告导出。

#### 3.4.2 性能指标定义

| 指标 | 单位 | 采集频率 | Android 数据源 | iOS 数据源 | HarmonyOS 数据源 |
|------|------|---------|---------------|-----------|-----------------|
| CPU 使用率 | % | 1s | `/proc/stat` + `top` | DTXConnection sysmontap | `hidumper --cpuusage` |
| 内存占用 | MB | 1s | `dumpsys meminfo` (PSS/USS) | DTXConnection sysmontap | `hidumper --mem` |
| FPS 帧率 | fps | 实时 | `dumpsys SurfaceFlinger --latency` | CADisplayLink / DTX | `hidumper -s RenderService` |
| Jank 卡顿 | 次/s | 实时 | `dumpsys gfxinfo framestats` | Core Animation | RenderService |
| 网络流量 | KB/s | 1s | `/proc/net/xt_qtaguid/stats` | DTX networking | `hidumper --net` |
| 电池电流 | mA | 5s | `dumpsys batterystats` | IOKit Battery | 电池服务 |
| 电池温度 | ℃ | 5s | `dumpsys battery` | IOKit Thermal | 温度服务 |
| GPU 使用率 | % | 1s | `dumpsys gfxinfo` | GPU Profiler | GPU 服务 |
| 线程数 | 个 | 2s | `/proc/<pid>/task` | DTX process info | 进程信息 |
| 启动耗时 | ms | 事件触发 | `am start -W` 输出 | Instruments | 启动日志 |

#### 3.4.3 功能需求

**FR-PT-001 测试会话管理**

- 创建测试会话：选择目标设备 + 目标应用 + 采集指标
- 支持为会话命名和添加标签（如"v2.1.0 首页优化后"）
- 支持同时对多个应用进行性能对比测试
- 会话自动保存，异常中断后可恢复
- 支持设置采集时长（1min / 5min / 15min / 30min / 自定义 / 无限制）

**FR-PT-002 实时数据可视化**

- 多指标折线图实时滚动展示（基于 ECharts）
- 支持指标叠加显示或独立面板
- 支持时间轴缩放和拖拽查看历史区间
- 关键事件标记（用户可手动打点标记，如"点击了购买按钮"）
- 超阈值区域高亮告警（如 CPU > 80% 标红）
- 支持暂停/恢复实时刷新

**FR-PT-003 阈值告警**

- 预设告警规则：

| 指标 | 默认阈值 | 告警级别 |
|------|---------|---------|
| CPU | > 80% | 警告 |
| CPU | > 95% | 严重 |
| 内存 | > 500MB | 警告 |
| 内存 | > 800MB | 严重 |
| FPS | < 50fps | 警告 |
| FPS | < 30fps | 严重 |
| Jank | > 3次/s | 警告 |

- 支持自定义阈值规则
- 告警时桌面通知 + 界面标记
- 告警历史记录可查询

**FR-PT-004 数据持久化与历史对比**

- 所有测试会话数据自动存储到本地 SQLite 数据库
- 支持按时间、设备、应用、标签筛选历史会话
- 支持选择 2-4 个历史会话进行指标对比（叠加折线图）
- 支持基线管理：设定某次测试为基线，后续测试自动与基线对比
- 数据保留策略：默认保留 90 天，可配置

**FR-PT-005 报告导出**

- 支持导出格式：HTML / PDF / CSV / JSON
- HTML 报告包含：
  - 测试概要（设备信息、应用版本、测试时长）
  - 各指标统计摘要（平均值、最大值、P95、P99）
  - 性能趋势图表（ECharts 静态图）
  - 告警事件列表
  - 关键截图（告警时刻自动截图）
- PDF 报告：HTML 报告的打印版本
- CSV/JSON：原始数据导出，便于二次分析
- 支持定时自动导出（测试结束后自动生成）

### 3.5 Midscene UI 自动化测试模块

#### 3.5.1 功能概述

基于 Midscene AI 框架，实现自然语言驱动的 UI 自动化测试。用户通过自然语言描述测试步骤，AI 自动识别界面元素并执行操作，大幅降低自动化测试编写门槛。

#### 3.5.2 核心原理

```
用户编写自然语言测试脚本
        ↓
Midscene 引擎解析指令
        ↓
截取当前设备屏幕截图
        ↓
多模态 AI 模型分析截图 + 指令
        ↓
返回目标元素坐标 / 提取信息 / 断言结果
        ↓
通过设备桥接层执行操作（tap/swipe/input）
        ↓
等待画面稳定，进入下一步
```

#### 3.5.3 功能需求

**FR-AT-001 测试脚本编辑器**

- 内置代码编辑器（基于 Monaco Editor）
- 支持 `.mts` (MobTestLab Script) 文件格式
- 语法高亮：自然语言指令 + JavaScript 控制流
- 智能提示：常用操作模板、历史指令补全
- 脚本模板：

```javascript
// 示例：登录流程测试
import { ai, aiAssert, aiQuery } from '@mobtestlab/midscene';

describe('登录流程', () => {
  it('应能成功登录', async () => {
    await ai('点击"我的"标签栏按钮');
    await ai('点击"登录/注册"按钮');
    await ai('在手机号输入框中输入 13800138000');
    await ai('点击"获取验证码"按钮');
    await ai('在验证码输入框中输入 123456');
    await ai('点击"登录"按钮');
    
    await aiAssert('页面显示了用户头像和昵称');
  });

  it('应提示手机号格式错误', async () => {
    await ai('在手机号输入框中输入 abc');
    await ai('点击"获取验证码"按钮');
    
    await aiAssert('页面显示"请输入正确的手机号"提示');
  });
});
```

**FR-AT-002 自然语言指令体系**

| 指令类型 | 语法示例 | 说明 |
|---------|---------|------|
| 点击操作 | `await ai('点击登录按钮')` | AI 识别"登录按钮"并点击 |
| 输入操作 | `await ai('在搜索框中输入 iPhone 16')` | AI 定位输入框并输入文本 |
| 滑动操作 | `await ai('向上滑动页面')` | AI 执行滑动手势 |
| 等待操作 | `await ai('等待加载完成')` | AI 判断页面是否加载完毕 |
| 断言操作 | `await aiAssert('购物车显示 3 件商品')` | AI 验证界面状态 |
| 查询操作 | `const price = await aiQuery('商品价格是多少')` | AI 提取界面信息 |
| 条件操作 | `await ai('如果有弹窗就关闭它')` | AI 条件判断并执行 |

**FR-AT-003 脚本管理**

- 项目级脚本组织：项目 → 模块 → 用例
- 支持文件夹树形结构管理
- 支持用例标签分类（冒烟测试、回归测试、P0/P1/P2）
- 支持用例搜索和筛选
- 支持用例导入/导出（JSON 格式）
- 支持 Git 版本管理集成（可选）

**FR-AT-004 测试执行**

- 单用例执行：选中一个用例点击运行
- 批量执行：选择多个用例或整个模块执行
- 执行队列：按顺序依次执行，支持失败后继续/停止策略
- 执行过程实时展示：
  - 当前执行步骤高亮
  - 每步操作前后截图对比
  - AI 识别区域标注（框选目标元素）
  - 执行耗时统计
- 支持设置步骤间等待时间（默认 1s）
- 支持设置单步超时时间（默认 30s）
- 支持执行中暂停/恢复/终止

**FR-AT-005 测试报告**

- 执行结果概览：通过/失败/跳过 数量和比例
- 失败用例详情：
  - 失败步骤截图
  - AI 识别结果说明
  - 期望 vs 实际对比
  - 错误堆栈信息
- 执行时间线：每步操作的时间轴视图
- 支持报告导出（HTML/PDF）
- 支持失败用例一键重试

**FR-AT-006 AI 模型配置**

- 支持配置 AI 模型提供商：
  - OpenAI (GPT-4V / GPT-4o)
  - Anthropic (Claude 4 Sonnet/Opus)
  - 百度文心大模型
  - 本地模型（Ollama 等）
- 支持配置 API Key 和 Endpoint
- 支持模型切换和对比测试
- 支持设置请求超时和重试策略
- Token 用量统计和成本估算

## 4. 非功能性需求

### 4.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 应用启动时间 | ≤ 3s | 冷启动到主界面可交互 |
| 投屏延迟 | ≤ 100ms | 从设备操作到画面更新 |
| 触控响应 | ≤ 50ms | 从鼠标点击到设备响应 |
| 性能数据采集 | ≤ 1s 间隔 | 不遗漏关键性能波动 |
| 内存占用 | ≤ 500MB | 单设备投屏 + 性能采集场景 |
| CPU 占用 | ≤ 15% | 稳态运行时（非录屏） |
| 多设备并发 | ≥ 4 台 | 同时投屏 + 性能采集 |

### 4.2 兼容性要求

**桌面端**：

| 操作系统 | 最低版本 | 架构 |
|---------|---------|------|
| macOS | 12.0 (Monterey) | x64 / ARM64 (Apple Silicon) |
| Windows | 10 (1903+) | x64 |
| Linux | Ubuntu 20.04+ | x64 |

**移动端**：

| 平台 | 最低版本 | 说明 |
|------|---------|------|
| Android | 7.0 (API 24) | scrcpy 最低要求 |
| iOS | 14.0 | WDA 兼容性要求 |
| HarmonyOS | 3.0 (API 9) | HDC 工具链要求 |

### 4.3 安全性要求

- ADB/HDC 连接仅限本地网络，不暴露公网端口
- AI 模型 API Key 加密存储（使用系统 Keychain / Credential Manager）
- 测试数据本地存储，不上传云端（除非用户主动配置）
- 应用自动更新使用 HTTPS + 签名验证
- 设备连接需用户确认授权（首次连接弹窗确认）

### 4.4 可靠性要求

- 设备断连自动重连（3 次重试，指数退避）
- 性能采集异常不影响投屏和交互功能
- 应用崩溃后自动恢复上次会话状态
- 数据库定期自动备份（每日）
- 日志系统：分级记录（debug/info/warn/error），支持导出排查

### 4.5 可扩展性要求

- 插件化架构：支持第三方性能采集插件
- 自定义指标：支持用户通过 Shell 命令自定义采集指标
- 脚本扩展：支持在 Midscene 脚本中调用自定义 JavaScript 函数
- 主题定制：支持亮色/暗色主题切换
